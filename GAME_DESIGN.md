# 踹他一百下 - 3D版游戏设计文档

## 一、游戏概述

**游戏名称**：踹他一百下

**游戏类型**：3D潜行/动作游戏

**技术栈**：Vue 3 + TypeScript + Three.js + Vite

**运行方式**：本地开发服务器，局域网可访问

---

## 二、游戏背景

有一个神人，每天提供花里胡哨的建议折磨你。你需要在他不注意时偷偷踹他，踹满一定次数即可过关。但要小心，如果被他发现，你会受到惩罚！

---

## 三、核心玩法

### 3.1 游戏目标

- 通过关卡制推进游戏
- 每关需要踹够指定次数才能过关
- 被发现3次（生命值归零）则游戏失败

### 3.2 关卡设计

| 关卡  | 需要踹的次数 | 累计 |
| ----- | ------------ | ---- |
| 第1关 | 5脚          | 5    |
| 第2关 | 10脚         | 15   |
| 第3关 | 20脚         | 35   |
| 第4关 | 35脚         | 70   |
| 第5关 | 50脚         | 120  |
| 第6关 | 70脚         | 190  |
| 第7关 | 100脚        | 290  |

## 四、操作说明

### 4.1 移动控制

| 按键 | 动作     |
| ---- | -------- |
| W    | 向上移动 |
| S    | 向下移动 |
| A    | 向左移动 |
| D    | 向右移动 |

### 4.2 其他操作

| 按键     | 动作                    |
| -------- | ----------------------- |
| 鼠标左键 | 踹击（靠近神人时）      |
| 空格     | 举起键盘挡脸（持续5秒） |
| 1/2/3    | 使用道具栏中的道具      |

### 4.3 视角

- 固定俯视角（45度）
- 相机自动跟随玩家移动
- 无需鼠标控制视角

## 五、神人AI系统

### 5.1 神人状态

```
┌──────────┐    随机触发    ┌──────────┐
│  正常状态  │ ──────────→ │ 手机闪烁  │
│ (用电脑)  │              │ (预警期)  │
└──────────┘              └──────────┘
      ↑                          │
      │                    3-5秒后
      │                          ↓
      │                    ┌──────────┐
      └─────────────────── │  回头状态  │
           动画完成后       │ (检查)    │
                          └──────────┘
```

### 5.2 回头动画流程

| 阶段 | 动作     | 时间  | 说明                   |
| ---- | -------- | ----- | ---------------------- |
| 1    | 站起     | 0.5秒 | 身体从椅子上升起       |
| 2    | 转身     | 0.4秒 | 身体旋转180度面向玩家  |
| 3    | 摇头观察 | 2秒   | 头部左右摇晃，检测玩家 |
| 4    | 转回     | 0.4秒 | 身体旋转回原位         |
| 5    | 坐下     | 0.5秒 | 身体下降回到椅子       |

### 5.3 预警机制

- 手机闪烁红色光效
- 屏幕边缘红色闪烁警告
- 状态栏显示"⚠️ 手机闪烁"
- 预警时间：3-5秒（随机）

### 5.4 检测规则

神人在"摇头观察"阶段会检测玩家：

- 玩家在躲藏点后方 → 安全
- 玩家使用键盘挡脸 → 安全
- 玩家使用隐身药水 → 安全
- 其他情况 → 被发现，扣血

## 六、玩家系统

### 6.1 角色外观

- 蓝色上衣
- 深蓝色裤子
- 黑色鞋子
- 肤色脸部，黑色眼睛，红色嘴巴
- 深棕色头发
- 左手始终拿着键盘

### 6.2 生命值

- 初始生命值：3条命
- 被发现一次扣1条命
- 生命值归零则游戏失败

### 6.3 键盘挡脸机制

- 按空格键举起键盘挡脸
- 持续时间：5秒
- 冷却时间：5秒（用完后）
- 头部上方显示倒计时进度条
- 进度条颜色变化：
  - 绿色：> 50%
  - 黄色：25%-50%
  - 红色：< 25%

## 七、场景设计

### 7.1 场景布局

```
┌─────────────────────────────────────────┐
│                 墙壁                     │
│  [盆栽]                    [文件柜]      │
│                                          │
│                                          │
│      [玩家出生点]                        │
│                                          │
│              [办公桌]                    │
│            [神人] [电脑] [手机]          │
│                                          │
│  [沙发]                                  │
└─────────────────────────────────────────┘
```

### 7.2 场景物体

| 物体   | 位置         | 作用                 |
| ------ | ------------ | -------------------- |
| 办公桌 | 场景后方中央 | 神人坐的位置         |
| 电脑   | 桌上         | 神人在使用           |
| 手机   | 桌上右侧     | 回头预警（闪烁红色） |
| 椅子   | 桌前         | 神人坐着             |
| 盆栽   | 左上角       | 躲藏点               |
| 文件柜 | 右上角       | 躲藏点               |
| 沙发   | 左下角       | 躲藏点               |

### 7.3 躲藏点

玩家站在躲藏点后方时，神人回头不会被发现。

## 八、道具系统

### 8.1 道具列表

| 道具     | 图标 | 效果           | 持续时间 |
| -------- | ---- | -------------- | -------- |
| 加速鞋   | 👟   | 移动速度x2     | 5秒      |
| 隐身药水 | 🧪   | 不被发现       | 5秒      |
| 噪音器   | 📢   | 吸引神人注意力 | 3秒      |
| 力量手套 | 🧤   | 踹击计数+2     | 5秒      |

### 8.2 道具机制

- 每10秒随机刷新1个道具
- 道具出现在场景边缘
- 玩家靠近自动拾取
- 最多携带3个道具
- 按 1/2/3 使用对应道具

## 九、UI界面

### 9.1 游戏头部

- 左侧：生命值（心形图标）
- 中间：当前关卡
- 右侧：踹击计数/目标次数

### 9.2 道具栏

- 位于游戏画面下方
- 显示已拾取的道具（最多3个）
- 显示道具快捷键（1/2/3）

### 9.3 状态栏

- 键盘状态（就绪/使用中/冷却中）
- 神人状态（用电脑/手机闪烁/回头检查）
- 躲藏状态（是否在躲藏点）

### 9.4 开始/结束界面

- 响应式设计，自适应屏幕大小
- 显示游戏说明和操作指南
- 显示最终得分

## 十一、技术架构

### 11.1 文件结构

```
src/
├── main.ts                    # 入口
├── App.vue                    # 根组件
├── components/
│   └── Game.vue              # 游戏主组件
├── game/
│   ├── SceneManager.ts       # Three.js场景管理
│   ├── Player.ts             # 玩家控制
│   ├── Enemy.ts              # 神人AI
│   ├── Props.ts              # 道具系统
│   ├── HidingSpots.ts       # 躲藏点
│   └── GameLoop.ts           # 游戏主循环
└── types/
    └── game.ts               # 类型定义
```

### 11.2 核心类

**SceneManager**：管理Three.js场景、相机、渲染器

**Player**：玩家控制、移动、动画、键盘挡脸

**Enemy**：神人AI、状态机、回头动画、检测逻辑

**Props**：道具生成、拾取、使用

**HidingSpots**：躲藏点管理、碰撞检测

**GameLoop**：游戏主循环、状态更新、胜负判定
