## Bug 2: 右键蓄力投掷无效

**诊断**: InputManager.ts 和 GameLoop.ts 的投掷逻辑代码正确。问题在于修复Bug 1后玩家初始没有装备武器，`canThrow()` 返回 false。玩家通过 Props 拾取武器并从背包装备后，右键投掷应正常工作。

**结论**: 无需改代码，修复Bug 1后验证即可。如仍不工作再排查。

## Bug 4: 游戏失败重试后不重置

**根因**: `GameLoop` 构造函数创建 `<canvas>` 作为局部变量追加到 container，但 `dispose()` 不会从 DOM 移除它。重试时新 canvas 追加到旧 canvas 旁边，旧 Babylon 引擎可能仍在渲染，导致画面停留在旧位置。

**文件**: `src/babylon/GameLoop.ts`

**改动 1**: canvas 保存为类属性（行60, 67-72）

```diff
   private _clickHandler: (() => void) | null = null
+  private canvas: HTMLCanvasElement | null = null

   constructor(...) {
-    const canvas = document.createElement('canvas')
-    canvas.style.width = '100%'
-    canvas.style.height = '100%'
-    container.appendChild(canvas)
-    this.engineContext = new EngineContext(canvas)
+    this.canvas = document.createElement('canvas')
+    this.canvas.style.width = '100%'
+    this.canvas.style.height = '100%'
+    container.appendChild(this.canvas)
+    this.engineContext = new EngineContext(this.canvas)
```

**改动 2**: dispose() 中移除 canvas（行408-422）

```diff
   dispose(): void {
     if (this._clickHandler) {
       document.removeEventListener('click', this._clickHandler)
     }
+    if (this.canvas?.parentNode) {
+      this.canvas.parentNode.removeChild(this.canvas)
+      this.canvas = null
+    }
     this.player.dispose()
     ...
```
