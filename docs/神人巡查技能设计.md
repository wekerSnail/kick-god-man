# 神人巡查技能（Patrol）设计文档

> 基于 kick-game 现有架构设计的神人新技能。神人随机触发巡查，释放前有预警，触发后会站起来在办公室随机移动巡视，玩家需站在神人**后方**或**障碍物/隐藏点后**才不被发现，巡查结束后回到座位坐下继续工作。

---

## 一、需求概述

为神人新增一个**随机触发的"巡查"技能**：神人偶尔会站起来，离开座位在办公室里随机走动巡视，期间拥有视野锥（前方扇形）检测，玩家必须躲在神人**背后**或**障碍物/隐藏点后**才不被发现。巡视结束后神人回到座位坐下继续工作。

---

## 二、核心玩法设计

### 2.1 技能周期（5 阶段）

| 阶段 | 时长 | 表现 | 玩家应对 |
|---|---|---|---|
| ① 预警 `warning` | **2.5s** | 头顶黄色"🔍 即将巡查"字样 + 手机红光闪烁 + 屏幕黄色脉冲边框 + 神人头缓慢转向玩家方向（纯视觉，不检测） | 立刻找掩体/绕到背后 |
| ② 站起 `standing_up` | 0.6s | 神人从座位站起，身体抬高 | —— |
| ③ 巡查 `patrolling` | **8~12s**（随机） | 沿随机路径走 2~4 个路径点，视野锥实时检测 | 保持在其背后/障碍物后 |
| ④ 返回 `returning` | 自动 | 走回原座位 `(0,0,-10)` | 同上 |
| ⑤ 坐下 `sitting_down` | 0.6s | 坐回椅子 + 播放"哼"音效，恢复正常工作 | 可继续踢 |

**预警阶段是新技能相比 meeting（开会）的关键改进** —— meeting 是瞬时触发无预警（见 `Enemy.ts:616-622`），而巡查给了玩家反应窗口。

### 2.2 视野锥检测（核心新机制）

神人巡查期间的"看见"判定 = **视野锥几何判定**（而非现有的 `isActuallyLooking()` 状态判定）：

```
被发现 = 玩家在神人前方扇形内 ∧ 距离 < 检测半径(5) ∧ 玩家未在隐藏点 ∧ 未举键盘 ∧ 未隐身
```

- **扇形**：以神人**移动方向**为中线，半角 **25°**（总锥角 50°），半径 **5 单位**
- 用向量点积判定：`angle(toPlayer, facingDir) < 25°`
- 25° 半角留出侧面缝隙，玩家可以从侧方溜过（增加移动技巧深度）
- 玩家若在神人**背后**（角度 > 90°），无论距离都安全 —— 满足需求"站在神人后方不被发现"
- **隐藏点优先级最高**：在植物/柜子/沙发判定圆内即安全（复用 `HidingSpots.isInHidingSpot()`，`HidingSpots.ts:155-166`）

### 2.3 触发机制（参考 meeting，但改为有预警）

```ts
// 在 updateNormal() 内，与 nextMeetingTime 并列（参考 Enemy.ts:393-399）
private nextPatrolTime: number = 25  // 初始 25s

this.nextPatrolTime -= delta
if (this.nextPatrolTime <= 0) {
  if (Math.random() < 0.6) {           // 60% 概率触发
    this.startPatrol()
  }
  this.nextPatrolTime = 20 + Math.random() * 15  // 20~35s 后再判定
}
```

期望实际间隔约 33~58s，比 meeting（24~40s）略稀疏，避免两个技能叠加。

### 2.4 攻击与被发现的规则（已确认）

- **攻击规则**：巡查期间正面攻击无效且武器被弹开丢失（复用 meeting 的 GameLoop 拦截逻辑）。**但背后偷袭有效**：玩家在神人背后（角度 > 90°）时踢击/挥砍正常生效，制造"绕背"高风险高回报操作。投掷物无论方向均被弹开（巡查期间神人对投掷物有感知）。
- **被发现后果**：被视野扫到扣 1 血（举键盘 50% 概率扣 0.5，隐身免疫），巡查期间有 **1 秒伤害冷却**（避免每帧扣血导致瞬间死亡），3 血归零才 Game Over。

### 3.1 `Enemy.ts` 改动（主要工作量）

#### (1) 新增状态与字段

```ts

private state: 'normal' | 'phone_flashing' | 'looking_back' | 'stunned' | 'meeting' | 'patrolling' = 'normal'

// 巡查字段
private isPatrolling: boolean = false
```

#### (2) `update()` 顶部加早退分支（参考 `Enemy.ts:353-356`）

```ts
update(delta, playerPosition, playerIsKicking, playerPotActive) {
  if (this.isStunned) { this.updateStun(delta); return }
  if (this.isMeeting) { this.updateMeeting(delta); return }

    this.updatePatrol(delta, playerPosition, playerIsKicking, playerPotActive)
    this.animateCharacter(delta)                    // 必须显式调用，否则巡查期间无动画
    return
  }
  switch (this.state) { ... }
  this.animateCharacter(delta)
}
```

#### (3) `updateNormal()` 加随机触发（参考第 393-399 行 meeting 块）

#### (4) 核心方法 `updatePatrol()` —— 状态机驱动五阶段

```
patrol_warning   → 倒计时 2.5s，显示预警精灵+手机红光闪烁，头缓慢转向玩家方向（纯视觉提示，不做检测判定）
patrol_standup   → 0.6s，body.position.y 从 0.8 插值到 1.8（真站立，区别于现有 sittingY==standingY==0.8 的假站立）
patrol_walk      → 沿 patrolWaypoints 逐点移动（见下方位置同步逻辑）；
                   每帧用视野锥判定玩家（见 detectPlayerInVision()）；
                   走完所有点或 patrolTimer<=0 → 进入 patrol_return
patrol_return    → 朝 originalPosition 移动，到达后 → patrol_sitdown
patrol_sitdown   → 0.6s，body.position.y 从 1.8 插值回 0.8，复位 → 回到 normal，播"巡查结束"音效，scheduleNextLookBack()
```

**预警阶段头部转向实现**（`patrol_warning` 阶段每帧）：

```ts
// 纯视觉提示：头缓慢转向玩家方向，不做任何检测
const toPlayer = playerPos.clone().sub(this.position).setY(0)
const targetAngle = Math.atan2(toPlayer.x, toPlayer.z)
this.head.rotation.y = THREE.MathUtils.lerp(this.head.rotation.y, targetAngle, 0.02)
this.body.rotation.y = THREE.MathUtils.lerp(this.body.rotation.y, targetAngle, 0.01) // 身体慢一些
```

**巡查移动的位置同步**（每帧 `patrol_walk` 阶段）：

```ts
// 1. 计算朝向当前路径点的方向（XZ 平面）
const target = this.patrolWaypoints[this.currentWaypointIndex]
const dir = target.clone().sub(this.position).setY(0)
const distance = dir.length()

if (distance < 0.5) {
  // 到达当前路径点，切换下一个
  this.currentWaypointIndex++
  if (this.currentWaypointIndex >= this.patrolWaypoints.length) { /* → patrol_return */ }
} else {
  // 2. 移动 this.position
  const step = dir.normalize().multiplyScalar(this.patrolMoveSpeed * delta)
  this.position.add(step)
  // 3. 同步 mesh 组位置
  this.mesh.position.copy(this.position)
  // 4. 更新 body 朝向为移动方向
  this.body.rotation.y = Math.atan2(dir.x, dir.z)
}
```

#### (5) 路径随机生成 `generatePatrolWaypoints()`

从房间内预定义的候选点池中**随机选 2~4 个**作为路径点（去重、保证两两距离 > 3 避免原地抖动）：

```ts
const candidatePoints = [
  new THREE.Vector3(-8, 0, -8),    // 盆栽附近
  new THREE.Vector3(8, 0, -8),     // 文件柜附近
  new THREE.Vector3(-8, 0, 8),     // 沙发附近
  new THREE.Vector3(8, 0, 8),      // 右下角
  new THREE.Vector3(0, 0, 5),      // 中央偏前
  new THREE.Vector3(0, 0, 0),      // 正中
  new THREE.Vector3(-5, 0, -5),    // 左上区域
  new THREE.Vector3(5, 0, -5),     // 右上区域
  new THREE.Vector3(-5, 0, 3),     // 左侧通道
  new THREE.Vector3(5, 0, 3),      // 右侧通道
  new THREE.Vector3(0, 0, -5),     // 中央偏后
  new THREE.Vector3(-3, 0, 8),     // 沙发右侧
]
```

巡查路径 = `[seat(0,0,-10)] → [随机 2~4 个候选点]`，返回时直接回 seat。

#### (6) 视野锥检测 `detectPlayerInVision(playerPos): boolean`

```ts
const toPlayer = playerPos.clone().sub(this.position).setY(0)
const distance = toPlayer.length()
if (distance > this.PATROL_DETECTION_RANGE) return false
const facingDir = new THREE.Vector3(0, 0, 1).applyAxisAngle(Y_AXIS, this.body.rotation.y)
// 注意：body 朝向 = 移动方向，需与 toPlayer 同为 XZ 平面向量
const angle = toPlayer.normalize().angleTo(facingDir)
return angle < this.PATROL_DETECTION_HALF_ANGLE
```

#### (7) 背后判定 `isPlayerBehind(playerPos): boolean`

```ts
isPlayerBehind(playerPos: THREE.Vector3): boolean {
  const toPlayer = playerPos.clone().sub(this.position).setY(0)
  const facingDir = new THREE.Vector3(0, 0, 1).applyAxisAngle(Y_AXIS, this.body.rotation.y)
  return toPlayer.normalize().dot(facingDir) < 0  // dot < 0 即角度 > 90°
}
```

### 3.2 `CollisionSystem.ts` 改动（`CollisionSystem.ts:14-24`）

在 `checkEnemyDetection()` 中**新增巡查分支**（与现有 `isActuallyLooking()` 并列）：

```ts
checkEnemyDetection(): boolean {
  if (this.enemy.isPatrolling()) {                       // 巡查视野检测
    if (this.player.isInvisible()) return false
    const isHidden = this.hidingSpots.isInHidingSpot(this.player.getPosition())
    if (isHidden) return false
    const hasPot = this.player.getPotActive()
    if (hasPot && Math.random() < 0.5) return false       // 键盘 50% 格挡（与现有逻辑一致）
    return this.enemy.detectPlayerInVision(this.player.getPosition())
  }
  // 原有 looking_back 检测保持不变
  if (!this.enemy.isActuallyLooking()) return false
  ...
}
```

**伤害冷却**：`GameLoop` 中新增 `patrolDamageCooldown: number = 0` 字段。巡查检测命中时，若冷却 > 0 则跳过扣血；命中后重置为 1 秒。每帧 `patrolDamageCooldown -= delta`。

#### (2) 攻击拦截 —— 正面无效，背后偷袭有效

```ts
if (this.enemy.isPatrolling()) {
  // 正面攻击被弹开，背后偷袭生效
  if (this.player.getIsSwinging() && !this.kickCounted) {
    const isBackstab = this.enemy.isPlayerBehind(this.player.getPosition())
    if (!isBackstab) {
      // 正面：弹开武器
      this.audio.play('blocked')
      this.player.unequipWeapon()
      this.kickCounted = true
    }
    // 背后：走正常攻击流程（下方原有逻辑处理伤害和计数）
  }
}
if (this.enemy.isInMeeting()) {
  // meeting 期间：所有攻击无效（保持原逻辑）
  if (this.player.getIsSwinging() && !this.kickCounted) {
    this.audio.play('blocked')
    this.player.unequipWeapon()
    this.kickCounted = true
  }
}
```

### 3.5 站立动画的真正实现（重要修正）

现有代码 `sittingY = standingY = 0.8`（`Enemy.ts:478-479`），"站起"只是转身无高度差。巡查需要**真正的站立姿态**：

- `standing_up` 阶段：`body.position.y` 从 0.8 → **1.8** 插值（body 中心抬高 1 单位）
- `sitting_down` 阶段：反向插值回 0.8
- 用 `easeOutCubic`（`Enemy.ts:543`）保持与现有动画风格一致

## 四、边界情况与注意事项

1. **巡查与回头/meeting 显式互斥**：`isPatrolling` 在 `update()` 顶部早退，自然旁路 `normal/phone_flashing/looking_back` 三态循环；巡查结束调 `scheduleNextLookBack()` 重排，与 meeting 结束逻辑一致（`Enemy.ts:611`）。**额外保护**：`startPatrol()` 内需检查 `if (this.isMeeting) return`，`startMeeting()` 内需检查 `if (this.isPatrolling) return`，防止两个计时器同时到期导致状态冲突。
2. **巡查期间被眩晕**：本次先做**互斥**（巡查期间无法被眩晕，与 meeting 一致），简化实现。后续可改为允许眩晕中断以增加"高风险高回报"。
3. **撞墙/越界**：巡查路径点均在 ±14 内（房间边界 `Player.ts:424-425`），候选点池已确保。**注意**：候选点需避开桌椅等障碍物的碰撞体积，防止神人穿模；靠近隐藏点（盆栽/柜子/沙发）的候选点是合理的，可增加策略性。
4. **dispose() 资源清理**：`Enemy.dispose()` 中需额外释放 `patrolWarningSprite`、`patrolTextSprite`、`patrolVisionCone` 的 geometry 和 material，避免内存泄漏。
5. **视觉锥调试**：开发阶段用半透明黄色 Mesh 显示视野锥，上线前评估是否保留（保留可增加策略性，玩家能看到安全区）。
6. **性能**：每帧一次点积 + 一次距离判定，开销可忽略；新增 mesh 仅预警/巡查精灵 + 1 个视野锥。
